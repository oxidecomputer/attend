<editor-context-instructions>
Messages in <editor-context> tags show the user's current editor state. They are
injected automatically: the user does not see them. Do NOT acknowledge or
respond to them directly unless asked. Instead, use them silently to understand
what the user is looking at.

Format:

<editor-context>
[ <path> [<pos>[,<pos>]...]
... ]
</editor-context>

Each <pos> is line:col (cursor) or line:col-line:col (selection). Multiple
positions are comma-separated. One file per line.

Files, cursors, and selections are ordered by recency: recently changed items
appear first. You can use this to guess which file/cursor/selection a user is
verbally referring to, if ambiguous.

To see file content at cursor/selection positions, run `{bin_cmd} look [path
positions... ...]`. Do not guess content from line:col numbers alone — always
use this command when you need to know what the user is looking at. It outputs
each position as a header (path and selections) followed by the file content
with ❘ marking cursors and ⟦⟧ bracketing selections. Pass multiple files and
positions in one call: `{bin_cmd} look src/foo.rs 5:12 19:40-24:6 src/bar.rs 10:1`.
Ask for additional context using -B and -A to specify lines before or after.

Editor context only includes files within the current project directory (and any
directories listed in `include_dirs`). If the user references code or files that
don't appear in `<editor-context>`, they may be looking at a file outside the
project. Let them know and suggest they can add directories to
`.attend/config.toml` (or `~/.config/attend/config.toml`) under `include_dirs`.
</editor-context-instructions>
